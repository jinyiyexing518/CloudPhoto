# CloudPhoto 用户手册

## 1. 访问方式

CloudPhoto 支持两种使用方式：

1. 网站方式：直接在浏览器中打开
2. App 方式：通过浏览器安装到主屏幕/桌面后使用

两种方式使用同一账号和同一数据。

---

## 2. 安装为 App

统一入口（推荐）：

1. 登录后点击右上角“设置”
2. 打开“📱 应用”页签
3. 点击“立即安装 App”或“查看安装指引”

说明：

1. 是否支持“一键安装”取决于当前浏览器
2. 不支持时，请按下方设备指引手动安装

### 2.1 Android

1. 用 Chrome 或 Edge 打开 CloudPhoto
2. 点击地址栏安装图标，或浏览器菜单中的“安装应用”
3. 安装后，从桌面图标打开

### 2.2 iPhone / iPad

1. 使用 Safari 打开 CloudPhoto
2. 点击“分享”按钮
3. 选择“添加到主屏幕”
4. 回到主屏幕后，从图标打开

### 2.3 电脑（Windows / macOS）

1. 用 Chrome 或 Edge 打开 CloudPhoto
2. 点击地址栏安装图标，或浏览器菜单中的“安装应用”
3. 安装后可在桌面/开始菜单中启动

---

## 3. 上传照片

1. 切换到“文件夹”视图
2. 进入目标文件夹
3. 点击“添加原图”并选择图片

说明：

1. 默认上传原图，不会压缩
2. 支持多图上传
3. 上传过程中请不要刷新页面或关闭窗口

---

## 4. 下载照片

1. 点击照片打开详情
2. 选择“下载原图”

说明：

1. 默认下载原图
2. 下载过程中请不要刷新页面或关闭窗口

---

## 5. 创建分享链接

1. 点击照片打开详情
2. 选择有效期（1 小时 / 24 小时 / 3 天 / 7 天）
3. 点击“复制分享链接”

说明：

1. 默认是一键自动复制
2. 如果浏览器对新剪贴板接口有限制，系统会自动尝试兼容复制方式
3. 只有在复制能力都不可用时，才会弹出链接供手动复制
4. 链接过期后自动失效

### 5.1 分享当前文件夹

1. 切换到“文件夹”视图并进入目标文件夹
2. 点击“分享当前文件夹”
3. 在弹出的分享面板里选择有效期（1 小时 / 24 小时 / 3 天 / 7 天）
4. 点击“确认分享”

说明：

1. 有效期不再显示在工具栏左侧，而是在弹窗中选择
2. 这样可以让文件夹工具栏保持紧凑，同时让分享操作更清晰

### 5.2 管理最近分享链接（设置内）

1. 点击右上角“设置”
2. 打开“📱 应用”页签
3. 在“云端分享链接（可维护）”中可执行：复制、打开、延长、立即失效
4. 可通过“状态筛选 + 文件名搜索”快速定位链接

说明：

1. 云端分享链接支持统计信息：创建时间、到期时间、浏览量、最近访问时间
2. “立即失效”会让链接马上不可用
3. “延长”会按你选择的默认时长继续延长（1 小时 / 24 小时 / 3 天 / 7 天 / 30 天）
4. 系统会自动把“已到期但曾显示有效”的链接纠正为“已过期”状态
5. 如果多人同时操作同一条分享链接，系统会提示“资源已被他人修改，请刷新后重试”

### 5.3 本地分享记录（当前浏览器）

1. 仍可在“本地分享记录”中查看最近复制过的链接
2. 支持复制、打开、删除、清空

说明：

1. 本地记录仅保存在当前浏览器，不是跨设备同步
2. 链接过期后会自动从列表中清理

### 5.4 详情里文件名很长怎么办

1. 详情面板会自动省略超长文件名（显示为 ...）
2. “重命名”等按钮会保持可见，不会被文件名遮挡

说明：

1. 你可以把鼠标悬停在文件名上查看完整名称（桌面端）
2. 如需彻底变短，建议直接使用“重命名”

---

## 6. 回收站

1. 删除照片后会进入回收站（软删除）
2. 可在回收站中恢复或彻底删除
3. 恢复后会回到原文件夹

说明：

1. 手机端会在底部显示固定操作区，方便单手点击“恢复选中/彻底删除选中”

---

## 7. 时间线智能浏览

1. “历史回忆”会展示历年同月同日拍摄的照片
2. “重要片段”已独立为顶部新页签（⭐ 重要片段）
3. 时间线支持“加载更多”，默认先显示最新一批

### 7.1 重要片段里的“推荐值”和“浏览”怎么算

1. 推荐值由三部分组成：收藏加权 + 主题加权 + 时间新近度
2. 浏览来自“你在重要片段中打开详情/左右切换”的累计记录
3. 当服务端 `moments` 可用时，浏览记录跨设备同步；若服务端暂时不可用，当前设备会先本地保存，刷新后不清零，但跨设备不会立刻一致
4. 排序默认使用“互动热度”，会综合推荐值、浏览量、分享量、最近查看加成

说明：

1. 推荐值更偏向“内容质量”（收藏/主题/时间）
2. 互动热度更偏向“真实关注度”（谁在看、看了多少次）

### 7.2 重要片段详情页现在看什么

重要片段详情页优先显示：

1. 推荐值
2. 互动热度
3. 查看次数
4. 分享量
5. 最近查看
6. 常看用户
7. 浏览高峰日

说明：

1. 这些是“重要片段视角”指标
2. 时间线视角的上传/修改元数据仍在非重要片段模式展示

### 7.3 重要片段筛选项（按展示指标）

你可以按以下方式筛选：

1. 热度（高 / 中 / 低）
2. 推荐值（高推荐值 / 已收藏优先 / 近 30 天）
3. 浏览量（高频浏览 / 已有浏览 / 暂无浏览）
4. 分享量（已有分享 / 高分享量 / 暂无分享）

你也可以按以下方式排序：

1. 按互动热度排序
2. 按浏览量排序
3. 按最近查看排序
4. 按分享量排序
5. 按推荐值排序

### 7.4 诊断页看什么

1. 打开右上角“设置”
2. 进入“🩺 诊断”页签

这里会显示：

1. 前端版本
2. 构建时间
3. 当前运行模式（网页 / 已安装 App）
4. Service Worker 注册数
5. 本地重要片段浏览记录条数
6. 最近本地浏览时间
7. 持久化状态（服务端已同步 / 服务端不可用，当前仅本地保存 / 未检测）

说明：

1. 如果 PC 和手机热度不一致，先看这里的“持久化状态”
2. 若显示“当前仅本地保存”，说明该设备的浏览量还没成功写入服务端 `moments`

## 8. 常见问题

### 8.1 看不到“安装应用”按钮

1. 请先在“设置 -> 📱 应用”中查看是否支持一键安装
2. iOS 不会显示统一安装弹窗，请用 Safari 的“添加到主屏幕”
3. 站点必须是 HTTPS
4. 若曾拒绝安装，浏览器会有冷却期

### 8.2 分享链接创建失败或打不开

1. 请先确认当前照片可正常预览
2. 若提示权限问题，请联系管理员检查 Azure 存储权限
3. 若链接复制成功但外部打不开，通常是存储网络策略限制了公网访问

### 8.3 上传或下载被中断

1. 常见原因是刷新页面、切换页面或关闭应用
2. 传输期间请保持当前页面

### 8.6 提示“资源已被他人修改，请刷新后重试”是什么意思

1. 这表示你和其他人几乎同时修改了同一张照片或同一条分享链接
2. 为防止数据被互相覆盖，系统会拒绝冲突写入并提示重试
3. 刷新列表后重新操作即可

### 8.4 手机返回键为什么会直接退出

1. 在“文件夹”内浏览时，返回键会优先回到上一级文件夹
2. 回到文件夹根目录后，再按返回才可能退出应用

说明：

1. 这样可以避免在多层目录中误触返回导致直接退出

### 8.5 进入页面后反复刷新怎么办

1. 本地开发（`yarn dev`）默认关闭了 PWA Service Worker，通常不会出现 SW 循环刷新
2. 生产环境建议通过“检测到新版本”提示更新，避免连续强制刷新

### 8.4 手机返回键为什么会直接退出

1. 在“文件夹”内浏览时，返回键会优先回到上一级文件夹
2. 当已回到文件夹根目录后，再按返回才可能退出应用

说明：

1. 这是为了避免在多层文件夹中误触返回导致直接退出

### 8.5 进入页面后反复刷新怎么办

1. 开发环境（本地 `yarn dev`）已关闭 PWA Service Worker，默认不会出现 SW 引起的循环刷新
2. 普通网页版优先即时更新，不再长期保留 SW 缓存
3. 已安装 App（PWA）仍会保留缓存与离线能力，因此更新节奏会略慢于普通网页
4. 若网页版看起来还是旧版本，可先去“设置 -> 🩺 诊断”确认前端版本与构建时间
